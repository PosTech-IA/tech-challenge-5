# Visão Geral do Projeto — Tech Challenge 5

## O que é o projeto

Sistema de processamento de documentos com análise por IA (Claude da Anthropic). O usuário faz upload de arquivos (PDF ou imagem), o sistema processa de forma assíncrona com um pipeline de IA e disponibiliza relatórios com os resultados da análise.

## Arquitetura

O projeto é um monorepo Python com arquitetura de microsserviços, gerenciado com [uv workspaces](https://docs.astral.sh/uv/). Todos os serviços rodam via Docker Compose.

```
tech-challenge-5/
├── services/
│   ├── gateway/     # API pública — autenticação JWT, rate limiting, roteamento
│   ├── upload/      # Recebe arquivos, armazena no MinIO, enfileira no Celery
│   ├── processor/   # Worker Celery — pipeline de análise com IA (Claude)
│   └── reports/     # Armazena e serve os relatórios gerados
├── shared/          # Schemas Pydantic e configurações compartilhadas
└── docker/
    └── postgres/    # Scripts de inicialização do banco
```

## Serviços

### Gateway (porta 8000)
- Único ponto de entrada da API
- Autenticação JWT via `pyjwt`
- Rate limiting via `slowapi`
- Faz proxy das requisições para `upload` e `reports`
- Stack: FastAPI + HTTPX

### Upload (porta 8001)
- Recebe arquivos PDF e imagens via multipart
- Armazena os arquivos no MinIO (object storage)
- Registra o status no PostgreSQL (`upload_db`)
- Enfileira tarefas de processamento no Celery/Redis
- Stack: FastAPI + SQLAlchemy + Celery + MinIO

### Processor (sem porta exposta — worker)
- Worker Celery que consome a fila do Redis
- Faz download do arquivo do MinIO
- Extrai texto de PDFs (`pypdf`) e imagens (`Pillow`)
- Envia o conteúdo para a API Claude (Anthropic) para análise
- Persiste o resultado no banco `processor_db`
- Stack: Celery + Anthropic SDK + SQLAlchemy

### Reports (porta 8002)
- Expõe os relatórios gerados pelo Processor
- Lê do banco `reports_db`
- Stack: FastAPI + SQLAlchemy + Celery

### Shared
- Pacote Python interno compartilhado entre todos os serviços
- Contém schemas Pydantic e configurações base (`pydantic-settings`)

## Infraestrutura (Docker Compose)

| Serviço    | Imagem            | Função                          |
|------------|-------------------|---------------------------------|
| postgres   | postgres:16-alpine | Banco de dados relacional (3 DBs: upload_db, processor_db, reports_db) |
| redis      | redis:7-alpine    | Broker de tarefas para o Celery |
| minio      | minio/minio       | Object storage para os arquivos |
| gateway    | build local       | Porta 8000 — entrada pública    |
| upload     | build local       | Porta 8001 — ingestão de arquivos |
| processor  | build local       | Worker assíncrono               |
| reports    | build local       | Porta 8002 — consulta de relatórios |

## Fluxo de dados

```
Cliente
  │
  ▼
Gateway (8000) — valida JWT, aplica rate limit
  │
  ├──▶ Upload (8001) — salva arquivo no MinIO, enfileira tarefa
  │         │
  │         ▼
  │      Celery (Redis) ──▶ Processor — baixa arquivo, chama Claude API, salva resultado
  │
  └──▶ Reports (8002) — retorna relatório ao cliente
```

## Stack tecnológica

| Categoria       | Tecnologias                                   |
|-----------------|-----------------------------------------------|

| IA              | Anthropic Claude (SDK `anthropic`)            |

| Linting         | Ruff                                          |
| Testes          | Pytest + pytest-asyncio                       |
| Gerenciador pkg | uv (workspaces)                               |
| Containers      | Docker + Docker Compose                       |

## Comandos úteis

```bash
make up      # Sobe todos os serviços em background
make down    # Derruba e remove volumes
make logs    # Acompanha logs em tempo real
make build   # Apenas builda as imagens
make test    # Roda os testes de todos os serviços
make lint    # Roda o Ruff em todo o código
```
